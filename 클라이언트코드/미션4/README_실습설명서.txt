[실습 목표]
- Fiddler를 통해 HTTP 요청/응답을 가로채고 쿠키 값을 조작해보자.
- 서버 코드에는 아무런 방어 로직이 없음 → 변조된 쿠키가 그대로 출력된다.
- 이후, 서버 코드를 수정하여 방어 로직을 추가한다.

[실습 순서]
1. Flask 서버 실행 후 브라우저에서 이름 입력 및 저장
2. Fiddler 실행 → 요청 가로채기 → Cookie 헤더 값 수정
3. 조작된 값이 서버에 반영되는지 확인
4. server.py에 방어 코드 삽입 (예: 길이 제한, 허용 문자 검사 등)
5. 다시 Fiddler로 시도 → 조작 실패를 확인
